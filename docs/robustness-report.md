# 交互输入鲁棒性评测报告

评测日期：2026-06-04  
评测目标：验证控制台交互在错误输入、空输入、越界输入、错误日期和外部数据更新场景下是否稳定。

## 评测结论

系统整体鲁棒性良好。常规菜单数字错误、空字符串、非法整数、非法小数、枚举输入错误和日期格式错误均不会导致程序崩溃。系统启动时会优先自动加载 `data/game-data.json`，因此外部维护 JSON 数据后无需修改 Java 代码即可生效。

## 已做加固

- 新增 `InputHelper.readDate()`，对日期输入进行循环校验。
- 新增启动自动加载逻辑：程序启动时优先加载 `data/game-data.json`；失败时回退到内置初始数据。
- 加强 `FileStorageService.load()`：外部 JSON 中枚举、日期或数值格式错误时，会包装成可读的 `IOException`，避免运行时异常直接打断菜单流程。
- 加强公开玩家查询：玩家 ID、用户名和昵称支持模糊候选；精确命中唯一玩家时直接进入操作；管理员和教练超级账户不会出现在公开检索结果中。
- 加强登录失败提示：管理员或教练账号失败登录时使用身份中性的失败文案，只有成功登录后才显示管理员身份。
- 新增派生数据一致性测试：验证玩家、英雄、装备增删后，相关动态展示不会残留旧引用。
- 新增中文姓名重叠检索测试：验证中文昵称部分重叠时，候选列表会返回所有相关玩家。

## 剩余边界

- 当前系统是单用户控制台程序，没有并发会话。
- JSON 解析器只支持本项目固定结构，不适合作为通用 JSON 解析库。
- 外部运营更新需要维护 ID 关系。例如装备要参与推荐，需要同时把装备 ID 加入对应英雄的 `compatibleEquipmentIds`；战队如要正式接收成员，需要同步更新玩家的 `teamId`。
