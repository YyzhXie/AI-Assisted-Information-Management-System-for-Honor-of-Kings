# Node.js 可视化版本说明

## 重要说明

本分支 `Node.js-visualization` 仅用于额外可视化实验和个人演示，不作为本 Java 课程作业提交内容。重新审视作业要求后，正式作业应以 `main` 分支中的纯 Java 控制台项目为准；`visualization/` 中的 Node.js 前端内容不计入课程作业范围。

## 目标

本分支在 Java 控制台项目基础上新增 Node.js 可视化版本。它复用 `data/game-data.json`，把之前测试过的玩家、战队、英雄、装备和对战数据展示为网页仪表盘，方便个人检查数据准确性和演示界面效果。

## 环境检查

本次实现前已检查本机 Node.js 环境：

- `node -v`：`v24.14.0`
- `npm -v`：`11.9.0`
- `where.exe node`：`D:\Node.js\node.exe`
- `where.exe npm`：`D:\Node.js\npm.cmd`

## 运行方式

该运行方式只适用于本分支的额外实验，不是课程作业运行方式。

在项目根目录执行：

```bash
cd visualization
npm test
npm start
```

启动后访问：

```text
http://localhost:3000
```

可选端口：

```bash
$env:PORT=3001
npm start
```

## 技术方案

- 后端：Node.js 标准库 `http`、`fs/promises`，不依赖 Express 或其他第三方库。
- 前端：原生 HTML、CSS、JavaScript 和 Canvas。
- 数据源：直接读取 `../data/game-data.json`。
- 接口：`GET /api/dashboard` 返回已经聚合、清洗后的公开可视化数据。
- 健康检查：`GET /api/health` 返回服务状态。

## 可视化内容

- 总览：玩家数、战队数、英雄数、装备数、对战记录数、平均等级。
- 图表：胜率排行榜、英雄类型分布、战队胜率进度条、装备评分 Top 8。
- 玩家：按 ID、用户名、昵称筛选，查看等级、胜率、英雄和最近对战。
- 战队：查看成员数、平均等级、胜率、顶尖玩家和成员列表。
- 英雄：查看属性、拥有者和推荐装备。
- 装备：查看基础评分、兼容英雄数量和综合分数。
- 对战：按战队筛选历史对战时间线。

## 数据与安全边界

可视化版本不向浏览器返回 `admins` 集合，也不会返回任何 `password` 字段。管理员 `admin` 和教练 `coach` 只保留在 Java 登录和本地 JSON 中，不进入公开可视化数据。

排行榜口径与 Java 版本保持一致：胜率优先时，按胜率降序；同位时按等级、对战次数和玩家 ID 顺序稳定排序。单局比赛只有胜负，不显示平局。

装备综合分数与 Java 版本保持一致：

```text
装备分数 = 基础评分 + 兼容英雄数量 * 3.5
```
