# 派生数据一致性多轮测试报告

评测日期：2026-06-05
评测目标：验证添加或删除玩家、英雄、装备后，战队成员、平均等级、总对战数、胜率、顶尖玩家、英雄拥有玩家情况、推荐装备和装备统计等动态信息是否按预期变化。

## 评测结论

通过。系统的派生信息主要由 `SearchService`、`RankingService`、`RecommendationEngine` 和 `GameDataManager` 从当前内存数据实时计算。多轮测试确认：

- 新增玩家会同步加入对应战队成员，并影响战队平均等级和顶尖玩家。
- 删除玩家会从战队成员列表中移除，并重新计算平均等级和顶尖玩家。
- 新增或删除玩家不会改变战队历史总对战数和战队历史胜率，因为这两项来自 `MatchRecord`。
- 新增英雄并分配给玩家后，英雄详情会显示拥有玩家；删除英雄后，玩家英雄列表会同步清理。
- 新增兼容装备后，英雄推荐装备和装备统计会实时变化；删除装备后，英雄兼容装备和推荐结果会同步清理。

## 测试方法

新增自动测试类：`src/test/DerivedDataConsistencyTest.java`。

测试使用 `data/game-data.json` 加载出的内存副本，不保存回正式 JSON 文件，因此不会污染正式数据集。测试命令：

```bash
javac -encoding UTF-8 -d out src/Main.java src/GuiMain.java src/model/*.java src/service/*.java src/util/*.java src/gui/*.java src/test/*.java
java -cp out test.DerivedDataConsistencyTest
```

实际输出：

```text
Derived data consistency test passed.
```

## 判断说明

战队总对战数和战队胜率由 `MatchRecord` 决定，不由成员列表直接决定。因此只添加或删除玩家时，战队成员、平均等级和顶尖玩家会变化，但战队历史总对战数和历史胜率应保持不变。只有添加、删除或修改对战记录时，这两项才应该变化。

英雄拥有玩家情况由 `Player.heroIds` 实时反查得到；推荐装备由英雄兼容装备列表和当前装备集合实时计算。因此删除英雄或装备时，必须清理对应引用。当前 `GameDataManager.deleteHero()` 和 `GameDataManager.deleteEquipment()` 已覆盖这些级联清理场景。
