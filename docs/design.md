# 设计文档

## 架构

项目分为三层：模型层、服务层和控制台菜单层。模型层保存数据和简单报表，服务层处理业务逻辑，`Main` 负责菜单流程。

## 排行榜规则

玩家胜率 = 胜场 / 总场次。胜率相同按等级降序排序，等级相同按 ID 升序排序，保证结果稳定。

## 装备统计公式

装备分数 = 基础评分 + 兼容英雄数量 * 2 + 使用次数 * 1.5。兼容英雄越多、在英雄配置中使用越多，排名越高。

## 推荐公式

推荐引擎先按英雄类型匹配装备类型，再按装备评分排序。坦克优先防御，法师优先法术，射手/刺客/战士优先攻击，辅助优先辅助装备。

## JSON 持久化

`FileStorageService` 保存固定结构 JSON：players、admins、heroes、equipment、teams、matches。加载时只解析本项目生成的格式，不作为通用 JSON 解析器。
