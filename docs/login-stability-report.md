# 连续登录稳定性评测报告

评测日期：2026-06-04  
评测目标：验证系统在连续登录多个同类型账号、不同类型账号以及错误登录后，是否能正确清理和切换当前登录状态。

## 数据集补充

原始数据集中已有 15 个玩家账号，但只有 1 个管理员账号。为了覆盖“连续登录多个同一种管理员账号”的稳定性，本次补充了第二个管理员账号。

|角色|用户名|密码|显示名|用途|
|---|---|---|---|---|
|管理员|admin|admin123|系统管理员|原始管理员账号|
|管理员|coach|coach123|战术教练|连续管理员登录测试|
|玩家|P001|123456|阿离同学|连续玩家登录测试|
|玩家|P002|123456|峡谷小周|连续玩家登录测试|
|玩家|P006|123456|法术核心|连续玩家登录测试|
|玩家|P012|123456|闪现向前|混合角色登录测试|

补充位置：

- `src/util/DataInitializer.java`
- `data/game-data.json`

## 评测方法

使用控制台输入流模拟真实用户连续操作。每次成功登录后，先在对应角色菜单选择 `0` 登出，再回到主菜单进行下一次登录。这样可以确认 `AuthenticationService.logout()` 是否将 `currentUser` 清空。

## 测试结果

### LS-01 连续登录多个玩家账号

输入顺序：`P001 -> 登出 -> P002 -> 登出 -> P006 -> 登出 -> 退出`

实际结果：

- `P001` 登录后进入玩家菜单，显示“欢迎 阿离同学”。
- 登出后回到主菜单。
- `P002` 登录后进入玩家菜单，显示“欢迎 峡谷小周”。
- 登出后回到主菜单。
- `P006` 登录后进入玩家菜单，显示“欢迎 法术核心”。

结论：通过。连续多个玩家账号不会沿用上一个玩家状态。

### LS-02 连续登录多个管理员账号

输入顺序：`admin -> 登出 -> coach -> 登出 -> 退出`

实际结果：

- `admin` 登录后进入管理员菜单，显示“欢迎 系统管理员”。
- 登出后回到主菜单。
- `coach` 登录后进入管理员菜单，显示“欢迎 战术教练”。

结论：通过。连续多个管理员账号可以正确切换。

### LS-03 管理员和玩家混合连续登录

输入顺序：`admin -> 登出 -> P012 -> 登出 -> coach -> 登出 -> admin错误密码 -> 退出`

实际结果：

- `admin` 正确进入管理员菜单。
- `P012` 正确进入玩家菜单。
- `coach` 正确进入管理员菜单。
- `admin/wrong` 显示登录失败，并停留在主菜单，没有进入任何旧账号菜单。

结论：通过。不同角色连续切换稳定，失败登录不会复用之前的登录状态。

### LS-04 从 JSON 加载后登录新增管理员

输入顺序：`加载JSON -> coach -> 登出 -> 退出`

实际结果：

- 系统显示“数据已从 data/game-data.json 加载”。
- `coach/coach123` 登录成功，并进入管理员菜单。

结论：通过。补充账号同时存在于初始化数据和 JSON 数据中，文件加载后仍可登录。

## 稳定性结论

当前登录稳定性良好。系统的登录状态只保存在 `AuthenticationService.currentUser` 中；成功登录会替换当前用户，登出会将其置空。连续登录同类账号、跨角色账号和错误密码场景均未发现状态串号、菜单权限残留或崩溃。

## 建议

如果后续扩展 GUI 或多用户并发，应将认证状态从单个 `currentUser` 改为会话对象。但当前控制台单用户模式下，现有实现稳定且足够符合课程要求。
